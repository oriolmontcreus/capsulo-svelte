## Capsules ã The Building Blocks

A **Capsule** is a self-contained folder that holds everything related to a content component: the schema, the UI component, and the auto-generated TypeScript types. Capsules are the fundamental building block of Capsulo.

### Anatomy of a Capsule

Take a Hero section as an example. You create a `hero/` folder with:

```
hero/
  hero.schema.tsx    ã defines the editable fields and their structure
  Hero.astro         ã the component responsible for rendering (can also be .tsx, .vue, .svelte)
  hero.types.ts      ã automatically generated by Capsulo's schema watcher (no manual work)
```

- **`hero.schema.tsx`** ã Defines every editable field for this component using the Fluent API
- **`Hero.astro`** (or `.tsx`, `.vue`, `.svelte`) ã The component responsible for rendering. Supports any frontend framework that Astro supports
- **`hero.types.ts`** ã Automatically generated by Capsulo's schema watcher. No manual work needed.

### Step-by-Step: How a Capsule Works

#### Step 1: Define the Schema

```tsx
import { createSchema } from "@/lib/form-builder/builders/SchemaBuilder";
import { Input, Textarea } from "@/lib/form-builder/fields";

export const HeroSchema = createSchema(
  "Hero",
  [
    Input("title")
      .label("Hero title")
      .description("The main headline")
      .required()
      .translatable()
      .defaultValue("Welcome to Capsulo"),

    Textarea("subtitle")
      .label("Subtitle")
      .rows(3)
      .translatable()
      .defaultValue("A content management system for developers"),
  ],
  "Hero section with title and CTA",
  "hero",
);
```

#### Step 2: Build the Component

Works with any framework supported by Astro:

**Astro:**
```astro
---
import type { SchemaProps } from '@/lib/schema-props';
import { HeroSchema } from './hero.schema';

export type Props = SchemaProps<typeof HeroSchema>;
const { title, subtitle } = Astro.props;
---
<section>
  <h1>{title}</h1>
  <p>{subtitle}</p>
</section>
```

**React:**
```tsx
import type { SchemaProps } from "@/lib/schema-props";
import { HeroSchema } from "./hero.schema";

type Props = SchemaProps<typeof HeroSchema>;

export function Hero({ title, subtitle }: Props) {
  return (
    <section>
      <h1>{title}</h1>
      <p>{subtitle}</p>
    </section>
  );
}
```

**Vue:**
```vue
<script setup lang="ts">
import type { SchemaProps } from "@/lib/schema-props";
import { HeroSchema } from "./hero.schema";

defineProps<SchemaProps<typeof HeroSchema>>();
</script>

<template>
  <section>
    <h1>{{ title }}</h1>
    <p>{{ subtitle }}</p>
  </section>
</template>
```

#### Step 3: Types Are Auto-Generated

`hero.types.ts` is automatically generated by Capsulo's schema watcher:

```typescript
// hero.types.ts ã auto-generated, do not edit
export interface HeroTypes {
  title: string;        // required Input ã string
  subtitle?: string;    // optional Textarea ã string | undefined
}
```

Add a toggle field and Capsulo knows it's a `boolean`. A required input becomes a `string`. An optional one becomes `string | undefined`. **Your schema IS your type definition.**

> **Schema Watcher**: The schema watcher runs automatically when you start your development server. It monitors your capsule folders and regenerates `.types.ts` files automatically whenever you update a schema.

## Schema System

The schema system is the heart of Capsulo. It uses a **Fluent API** pattern where each field is a builder that returns itself, allowing method chaining.

### `createSchema()`

Creates a new content schema that defines the structure of a Capsule:

```typescript
createSchema(
  name: string,          // Display name in the CMS
  fields: Field[],       // Array of field definitions
  description: string,   // Human-readable description
  slug: string,          // URL-friendly identifier
  icon?: ReactNode       // Optional icon for the CMS sidebar
)
```

### The Fluent API Pattern

Every field builder follows a fluent pattern. Methods like `.label()`, `.required()`, `.defaultValue()`, `.translatable()`, `.hidden()`, `.description()`, and `.colSpan()` are available on most fields:

```typescript
Input("title")
  .label("Hero Title")
  .description("The main headline of the hero section")
  .required()
  .translatable()
  .defaultValue("Welcome")
  .placeholder("Enter your title")
```

### Common Field Properties

| Property | Type | Default | Description |
|---|---|---|---|
| `label` | `string` | ã | Sets the field label displayed in the CMS |
| `description` | `string` | ã | Helper text displayed below the field |
| `required` | `boolean \| function` | `true` | Marks the field as required (can be conditional) |
| `defaultValue` | varies | ã | Sets the initial value |
| `translatable` | `boolean` | `false` | Enables multi-language support |
| `hidden` | `boolean \| function` | `false` | Hides the field conditionally |
| `showInTable` | `boolean` | `true` | Shows the field in the collection table view |
| `colSpan` | `number \| 'full' \| ResponsiveValue` | ã | Column span inside a Grid layout |
| `placeholder` | `string` | ã | Placeholder text (for applicable fields) |

## Field Types

Capsulo provides a rich set of field types for building content schemas:

### Input

A fundamental component for capturing text, numbers, emails, passwords, and other single-line values.

```typescript
import { Input } from "@/lib/form-builder/fields/Input/input.builder";

Input("fieldName").label("Field Label");
```

**Full API:**
| Option | Type | Default | Description |
|---|---|---|---|
| `label` | `string` | ã | Sets the label of the input field |
| `description` | `string` | ã | Adds helper description below the field |
| `placeholder` | `string` | ã | Sets placeholder text |
| `required` | `boolean \| function` | `true` | Marks as required (conditional support) |
| `defaultValue` | `string` | ã | Sets initial value |
| `type` | `'text' \| 'email' \| 'url' \| 'password' \| 'number'` | `'text'` | HTML input type |
| `prefix` | `ReactNode` | ã | Adds content to the start of the input |
| `suffix` | `ReactNode` | ã | Adds content to the end of the input |
| `minLength` | `number` | ã | Minimum character count |
| `maxLength` | `number` | ã | Maximum character count |
| `min` / `max` | `number` | ã | Numeric range (for number inputs) |
| `step` | `number` | ã | Step interval (for number inputs) |
| `allowDecimals` | `boolean` | `true` | Allows decimal numbers |
| `regex` | `string \| RegExp` | ã | Validates against a regex |
| `translatable` | `boolean` | `true` | Enables translation support |
| `showInTable` | `boolean` | `true` | Shows in collection table view |
| `hidden` | `boolean \| function` | `false` | Hides the field conditionally |
| `colSpan` | `number \| 'full' \| ResponsiveValue` | ã | Column span in Grid layout |

**Examples:**
```typescript
// Basic text input
Input("username").label("Username").placeholder("Enter your username");

// Email with icon prefix
Input("email")
  .label("Email Address")
  .type("email")
  .placeholder("john@example.com")
  .prefix(<Mail className="size-4" />);

// With global variable
Input("pageTitle").label("Title").defaultValue("Welcome to {{siteName}}");

// Number input with validation
Input("age").label("Age").type("number").min(0).max(150).allowDecimals(false);

// With regex validation
Input("slug").label("URL Slug").regex(/^[a-z0-9-]+$/).description("Only lowercase letters, numbers, and hyphens");
```

### Textarea

A multi-line text input field for longer content.

```typescript
import { Textarea } from "@/lib/form-builder/fields";
```

**Key options:**
| Option | Type | Description |
|---|---|---|
| `rows` | `number` | Sets the number of visible text lines |
| `translatable` | `boolean` | Enables translation support |
| All common options from Input | | label, description, required, defaultValue, etc. |

**Examples:**
```typescript
Textarea("description").label("Description").rows(4).translatable();
Textarea("bio").label("Biography").rows(6).maxLength(500).placeholder("Tell us about yourself...");
```

### Select

A versatile dropdown component for selecting from predefined options. Supports two modes: standard dropdown and searchable combobox.

```typescript
import { Select } from "@/lib/form-builder/fields/Select/select.builder";

Select("fieldName")
  .label("Field Label")
  .options([
    { label: "Option 1", value: "opt1" },
    { label: "Option 2", value: "opt2" },
  ]);
```

**Full API:**
| Option | Type | Default | Description |
|---|---|---|---|
| `options` | `SelectOption[]` | ã | Array of selectable options with label, value, prefix, suffix, description |
| `groups` | `SelectOptionGroup[]` | ã | Grouped options with group labels |
| `searchable` | `boolean` | `false` | Enables combobox search mode |
| `searchPlaceholder` | `string` | `'Search...'` | Search input placeholder |
| `emptyMessage` | `string` | ã | Message when no search results |
| `internalLinks` | `boolean` | `false` | Enables page selection mode from site pages |
| `prefix` / `suffix` | `ReactNode` | ã | Content before/after the select |

**Features:**
- Standard dropdown and searchable combobox modes
- Grouped options for organization
- Multi-column layouts for complex selections
- Internal links mode for selecting pages from your site
- Virtualization for large option lists
- Custom rendering per option
- Options support: label, value, prefix, suffix, description

**Examples:**
```typescript
// Basic select
Select("status").label("Status").options([
  { label: "Draft", value: "draft" },
  { label: "Published", value: "published" },
  { label: "Archived", value: "archived" },
]);

// Searchable select
Select("category").label("Category").searchable(true).options([...]);

// Grouped options
Select("timezone").label("Timezone").groups([
  { label: "Americas", options: [
    { label: "New York", value: "America/New_York" },
    { label: "Los Angeles", value: "America/Los_Angeles" },
  ]},
  { label: "Europe", options: [
    { label: "London", value: "Europe/London" },
    { label: "Madrid", value: "Europe/Madrid" },
  ]},
]);

// Internal links mode
Select("linkToPage").label("Link to Page").internalLinks(true);
```

### DateField

A component for capturing dates and date ranges with multiple display modes.

```typescript
import { DateField } from "@/lib/form-builder/fields";

DateField("fieldName").label("Field Label");
```

**Full API:**
| Option | Type | Default | Description |
|---|---|---|---|
| `mode` | `'single' \| 'range'` | `'single'` | Single date or date range selection |
| `variant` | `'calendar' \| 'input'` | `'calendar'` | Calendar popup or direct input mode |
| `format` | `string` | ã | Date display format |
| `captionLayout` | `'label' \| 'dropdown'` | `'label'` | Month/year navigation style |
| `showYearDropdown` | `boolean` | `false` | Shows year dropdown |
| `showMonthDropdown` | `boolean` | `false` | Shows month dropdown |
| `yearRange` | `(from: number, to?: number) => this` | ã | Sets year range for dropdown |
| `fixedWeeks` | `boolean` | `false` | Always shows 6 weeks to prevent layout shifts |
| `showOutsideDays` | `boolean` | `false` | Shows days from adjacent months |
| `weekStartsOn` | `0 \| 1 \| 2 \| 3 \| 4 \| 5 \| 6` | ã | Which day the week starts on (0=Sunday) |
| `locale` | `string` | ã | Locale for date formatting |
| `placeholder` | `string` | ã | Placeholder text |
| `showInTable` | `boolean` | `true` | Shows in collection table view |
| `hidden` | `boolean \| function` | `false` | Conditional hiding |
| `colSpan` | `number \| 'full' \| ResponsiveValue` | ã | Column span in Grid layout |

**Examples:**
```typescript
// Basic date
DateField("dob").label("Date of Birth").placeholder("Select your birth date");

// Date range
DateField("eventDates").label("Event Dates").mode("range").description("Select start and end dates");

// Input variant with long format
DateField("departure").label("Departure Date").variant("input").format("long");

// Dropdown with year/month selectors
DateField("anniversary")
  .label("Anniversary")
  .captionLayout("dropdown")
  .showYearDropdown(true)
  .showMonthDropdown(true)

  .label("Booking Date")
  .fixedWeeks(true)
  .showOutsideDays(true)
  .description("Calendar always shows 6 weeks to prevent layout shifts");
```

### FileUpload

Enables users to upload and manage files including images, documents, and media. Supports multiple variants, file validation, image optimization, and convenient presets.

```typescript
import { FileUpload } from "@/lib/form-builder/fields/FileUpload/fileUpload.builder";

FileUpload("fieldName").label("Field Label");
```

**Full API:**
| Option | Type | Default | Description |
|---|---|---|---|
| `accept` | `string` | ã | Accepted file types (MIME types or extensions) |
| `maxSize` | `number` | ã | Maximum file size in bytes |
| `maxFiles` | `number` | ã | Maximum number of files allowed |
| `multiple` | `boolean` | `false` | Allows multiple file selection |
| `variant` | `'list' \| 'grid' \| 'inline'` | `'list'` (or `'inline'` for single mode) | Display variant |
| `inlineConfig` | `InlineConfig` | ã | Configuration for the inline variant |
| `imageOptimization` | `ImageOptimizationConfig` | ã | Image optimization settings |
| `defaultValue` | `FileUploadValue` | ã | Initial value |
| `required` | `boolean \| function` | `true` | Marks as required |
| `showInTable` | `boolean` | `true` | Shows in table view |
| `hidden` | `boolean \| function` | `false` | Conditional hiding |
| `colSpan` | `number \| 'full' \| ResponsiveValue` | ã | Column span in Grid layout |

**Preset methods:**
- `.images()` ã Configures for image upload (accepts image MIME types)
- `.documents()` ã Configures for document upload
- `.avatar()` ã Preconfigured for avatar images (square, single file, inline)
- `.coverImage()` ã Preconfigured for cover/banner images
- `.aspectRatio(ratio)` ã Enforces aspect ratio (e.g., "square", "16:9")

**Examples:**
```typescript
// Basic image upload
FileUpload("heroImage").label("Hero Image").images().maxSize(5 * 1024 * 1024);

// Document upload with type restriction
FileUpload("document").label("PDF Document").documents().accept(".pdf");

// Avatar with square aspect ratio
FileUpload("avatar")
  .label("Profile Picture")
  .variant("inline")
  .aspectRatio("square")
  .images()
  .maxSize(2 * 1024 * 1024);

// Multiple images in grid layout
FileUpload("gallery")
  .label("Photo Gallery")
  .images()
  .multiple()
  .maxFiles(10)
  .variant("grid");

// Cover image preset
FileUpload("banner").label("Cover Banner").coverImage();
```

## Layout System

Layouts control how fields are arranged within the CMS form. They are themselves field builders that can contain other fields.

### Grid Layout

Organize fields in a responsive grid system with breakpoint-based column configuration.

```typescript
import { Grid } from "@/lib/form-builder/layouts";

Grid({ base: 1, md: 2 }).contains([
  Input("firstName").label("First Name"),
  Input("lastName").label("Last Name"),
])
```

**Column configuration:**
| Option | Type | Description |
|---|---|---|
| `base` | `number` | Columns at base/mobile breakpoint |
| `sm` | `number` | Columns at small breakpoint |
| `md` | `number` | Columns at medium breakpoint |
| `lg` | `number` | Columns at large breakpoint |
| `xl` | `number` | Columns at extra-large breakpoint |

**Column spanning:**
Fields inside a Grid can use `.colSpan()` to span multiple columns:
```typescript
Grid({ base: 1, md: 3 }).contains([
  Input("title").colSpan(2),           // Spans 2 columns
  Input("subtitle").colSpan(1),        // 1 column
  Input("wideField").colSpan("full"),  // Full width (all columns)
])
```

### Tabs Layout

Organize fields into tabbed sections for better organization of complex forms.

```typescript
import { Tabs } from "@/lib/form-builder/layouts";

Tabs([
  { label: "Content", fields: [
    Input("title").label("Title"),
    Textarea("body").label("Body"),
  ]},
  { label: "SEO", fields: [
    Input("metaTitle").label("Meta Title"),
    Textarea("metaDescription").label("Meta Description"),
  ]},
  { label: "Advanced", fields: [
    Switch("published").label("Published"),
    Select("layout").label("Layout").options([...]),
  ]},
])
```

### Configuration

Enable translations in `capsulo.config.ts`:

```typescript
const config: CapsuloConfig = {
  i18n: {
    defaultLocale: "en",         // Default locale for content
    locales: ["en", "es", "fr"], // Available locales for translation
    fallbackLocale: "en",        // Fallback when translation is missing
  },
};
```

> Locale codes are validated against ISO 639-1 standards (e.g., `"en"`, `"es"`, `"fr"`, or extended formats like `"en-US"`). Translations are automatically enabled when you configure the `i18n` object.

### Adding Translatable Fields

Mark fields as translatable using `.translatable()`:

```typescript
Input('heroTitle').label('Hero Title').required().translatable().defaultValue('Welcome')
Textarea('metaDescription').label('Meta Description').rows(3).translatable()
RichEditor('content').label('Page Content').translatable()
```

> Only text-based fields support translations: **Input**, **Textarea**, and **RichEditor**. Non-text fields (Select, Switch, DateField, ColorPicker, FileUpload, Repeater) do NOT support `.translatable()`.

### Data Storage Format

When a field is marked as translatable, its value is stored as a locale-keyed object:

```json
{
  "heroTitle": {
    "type": "input",
    "translatable": true,
    "value": {
      "en": "Welcome to our site",
      "es": "Bienvenido a nuestro sitio",
      "fr": "Bienvenue sur notre site"
    }
  }
}
```

### Translation Status Indicators

In the CMS interface, translatable fields display a **Languages icon** next to the label:
- **Green icon** ã All locales have translations
- **Red icon** ã One or more locales are missing translations

### Translation Sidebar

When you focus a translatable field in the CMS:
1. The sidebar automatically opens showing translation inputs for each non-default locale
2. Input fields are pre-populated with existing translations
3. Changes are automatically synced to the main form data
4. RichEditor fields open in a dialog for better editing experience
5. Close by clicking the X button or clicking outside the translatable field

### Using Translations in Frontend

**Astro Component (recommended):**
```astro
---
import { loadPageData, getAllComponentsData } from '@/lib/cms-loader';

const locale = Astro.currentLocale || 'en';
const pageData = await loadPageData('index');

const title = heroData?.heroTitle;  // Already resolved for the locale
---

  <h1>{title}</h1>
</section>
```

**React Component:**
```tsx
import { getGlobalVar } from '@/lib/cms-loader';

function MyComponent() {
  const [globals, setGlobals] = useState(null);
  useEffect(() => {
    getGlobalVar('es').then(setGlobals);  // Pass locale for localized values

```

### URL-based Locale Detection

```typescript
import { getLocaleFromPathname, isValidLocale } from '@/lib/i18n-utils';

const locale = getLocaleFromPathname('/es/about');  // Result: 'es'
const defaultLocale = getLocaleFromPathname('/about');  // Result: 'en' (defaultLocale)
```

### Fallback Behavior

When a translation is missing for a locale:
1. If `fallbackLocale` is configured, that value is used
2. Otherwise, `defaultLocale` value is used
3. For complex nested structures (repeaters, objects), a deep merge is performed between target and fallback values

| Function | Description | Type |

### Advanced: Translation Store

For custom integrations, you can use the external store directly:

```typescript
import * as translationStore from '@/lib/form-builder/context/translation-store';

// Subscribe to changes

  const state = translationStore.getSnapshot();

// Get current component
const component = translationStore.getCurrentComponent();
```

> **Warning**: Direct store access is advanced usage. Prefer the React hooks (`useTranslationData`, `useTranslation`) for most use cases.

### Defining Global Variables

Edit `src/config/globals/globals.schema.tsx`:

```tsx
import { Input } from '@/lib/form-builder/fields';
import { createSchema } from '@/lib/form-builder/builders/SchemaBuilder';
import { Globe } from 'lucide-react';

export const GlobalsSchema = createSchema(
    'Global Variables',
    [
        Input('siteName')
            .label('Site Name')
            .description('The name of your website')
            .required()
            .translatable()
            .defaultValue('My Awesome Site'),

        Input('siteEmail')
            .label('Contact Email')
            .type('email')
            .description('Primary contact email address')
            .defaultValue('contact@example.com'),

        Input('companyName')
            .label('Company Name')
            .description('Your company or organization name')
            .translatable()
            .defaultValue(''),
    ],
    'Global site-wide settings',
    'globals',
    <Globe size={18} />,
);
```

> You can use any CMS field type (Input, Textarea, Select, Checkbox, Number, Date, FileInput, etc.). Use `.translatable()` to enable multi-language support for a field.

### Using Variables in CMS Fields

Any Input or Textarea field automatically supports global variable injection:

1. Type `{{` in any Input or Textarea field
2. Autocomplete menu appears with available variables
3. Navigate with arrow keys and select with Enter
4. Variable inserted as `{{variableName}}`

```typescript
Input('pageTitle').label('Page Title').defaultValue('Welcome to {{siteName}}')
Textarea('welcomeMessage').label('Welcome Message').defaultValue('Contact us at {{siteEmail}}')
```

### Using Variables in Frontend

**Astro Context (recommended):**
```astro
---
```

Variables in CMS content are automatically resolved when loaded:

**React Hook:**

## Auto Discovery

One of Capsulo's most powerful features is auto discovery. When you import a Capsule component into one of your pages, Capsulo detects it automatically and surfaces the right CMS fields for that page.

**No registration, no configuration, no "hey CMS, this page uses this component."**

```astro

// src/pages/index.astro
import Hero from '@/capsules/hero/Hero.astro';
```

Visit `/admin` and the CMS fields for `Hero` and `Features` are already there, scoped to the index page. You just build your website normally, and Capsulo figures the rest out.

## Configuration

All Capsulo configuration is centralized in `capsulo.config.ts`:

```typescript
interface CapsuloConfig {
  // GitHub repository for auth and content storage
  github: {
    owner: string;           // GitHub repository owner (username or org)
    repo: string;            // GitHub repository name
  };

  // Application metadata
  app: {
    name: string;            // Application name displayed in the CMS (default: "Capsulo CMS")
    version: string;         // Application version
    authWorkerUrl: string;   // Cloudflare Worker URL for OAuth
  };

  // CMS UI customization
  ui: {
    pageFilterRegex: string;           // Regex for which pages show in CMS sidebar
    contentMaxWidth: string;           // Default max width of content area
    autoSaveDebounceMs: number;        // Auto-save debounce time
    autoSaveBlockDurationMs: number;   // Auto-save block duration
  };

  // Internationalization
  i18n?: {
    defaultLocale: string;    // Default locale (e.g., "en")
    locales: string[];        // Available locales (e.g., ["en", "es", "fr"])
    fallbackLocale?: string;  // Fallback locale for missing translations
  };

  // Storage configuration
  storage?: {
    uploadWorkerUrl?: string; // URL for file upload worker (Cloudflare R2)
  };

  // Cache configuration
  cache?: {
    dbName?: string;          // IndexedDB database name for CMS storage
    dbVersion?: number;       // IndexedDB schema version (increment if store structure changes)
    cacheExpiryHours?: number; // Cache expiry time in hours (default: 24)
  };
}
```
